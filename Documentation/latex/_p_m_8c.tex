По полученным из \doxylink{_t_i_m__lib_8c_a43d2c695c558f49cb2df8c0745c28531}{Zero\+Cross\+Timings} данным находим бинарный код, путём подсчёта фазовых сдвигов. Фазовые сдвиги определяются исходя из времени переходов через ноль внутри фазовых переходов.

Логика демодуляции следующая\+:
\begin{DoxyItemize}
\item Считается нулевая фаза, т.\+е. фаза, относительно которой будут подсчитаны фазовые сдвиги. Фаза считается как остаток от деления времени перехода через 0 на период колебаний несущей, далее находится их среднее арифметическое, что и считается за нулевую фазу
\item Для остальных фазовых переходов фазовый находится как наиболее распространённое значение. Другими словами, среднее арифметическое заменяется на количество фаз, попадающих в промежуток. Например, если будут получены фазы 0 0 pi pi/2, то результатом будет 0, а не pi/2
\item По полученному фазовому сдвигу находится бинарный код, которому этот фазовый переход соответсвует. Так фазовый переход 0 соответсвует коду 00, pi/2 -\/ 10, pi -\/ 11, -\/pi/2 -\/ 01. Полученный результат суммируется с нулевой фазой и используется как нулевая фаза для следующего фазового перехода
\item После демодуляции всех 40 фазовых переходов, запускается поиск АРСФ кода \doxylink{_p_m_8c_ad2390af362a4e67b648e37d8f4203f8e}{Decoding} 
\end{DoxyItemize}\Hypertarget{_p_m_8c_a9f059a669299ce6cdba20cf4efbd6186}\label{_p_m_8c_a9f059a669299ce6cdba20cf4efbd6186}

Функция находит в принятом сигнале переходы через 0 \doxylink{_t_i_m__lib_8c_a43d2c695c558f49cb2df8c0745c28531}{Zero\+Cross\+Timings} и использует эту информацию для заполнения \doxylink{_t_i_m__lib_8c_ad14650db362e7e1e1d34b5a4084cc9f1}{End\+Of\+Sample}

Логика анализа следующая\+:
\begin{DoxyItemize}
\item Определяется переход через ноль (его тик, т.\+е. временной эквивалент)
\item Если между текущим и прошлым переходом через 0 ровно 8 тиков, то считается что измерения происходят внутри одного фазового перехода, увеличивается \doxylink{_p_m_8c_a073bdb7c297960b4d9332ce4d89d1eca}{Through\+Period\+Count}
\item Проверка на \doxylink{_p_m_8c_a1f945a1b7b1640ec42ad649537247fa2}{Period\+Continues}\+: если Period\+Continues == 0, то есть период не продолжается (или другими словами, предыдущий фазовый переход закончился), тогда, если пройдена проверка, информация о фазовом переходе записывается в \doxylink{_t_i_m__lib_8c_ad14650db362e7e1e1d34b5a4084cc9f1}{End\+Of\+Sample}, увеличивается \doxylink{_p_m_8c_a9f189c2f26f1ad1b9de5ec6d79fe514a}{Sample\+Counter}. Проверка заключается в определении Period\+Count для текущего фазового перехода. Учитывая, что в фазовом переходе должно быть 16 периодов, то можно предположить, что при Period\+Count \texorpdfstring{$<$}{<} 6 анализ был произведён неверно, текущий фазовый переход ложный. В таком случае, информация о Period\+Count удаляется из \doxylink{_p_m_8c_a073bdb7c297960b4d9332ce4d89d1eca}{Through\+Period\+Count}
\item Если в течении 15 периодов не был обнаружен фазовый переход, то считается, что он произошел с нулевым изменением фазы
\item Если между текущим и прошлым переходом через 0 не 8 тиков, то считается, что текущий фазовый переход завершился, \doxylink{_p_m_8c_a1f945a1b7b1640ec42ad649537247fa2}{Period\+Continues} = 0
\item Когда \doxylink{_p_m_8c_a9f189c2f26f1ad1b9de5ec6d79fe514a}{Sample\+Counter} \texorpdfstring{$>$}{>} 43 или \doxylink{_p_m_8c_a073bdb7c297960b4d9332ce4d89d1eca}{Through\+Period\+Count} \texorpdfstring{$>$}{>} 700, функция заканчивает работу и вызывает \doxylink{_p_m_8c_ad661cb7c42214740f919cf580a37ffec}{Analysis\+End()}
\item Если за время анализа \doxylink{_t_i_m__lib_8c_a7558533ab0553c6f22905cb4a83f93c3}{Voltages\+Data} предыдущий пункт не выполнился, то последняя точка \doxylink{_t_i_m__lib_8c_a7558533ab0553c6f22905cb4a83f93c3}{Voltages\+Data} считается за конец последнего фазового перехода, вызывается \doxylink{_p_m_8c_ad661cb7c42214740f919cf580a37ffec}{Analysis\+End()} 
\end{DoxyItemize}
